# Система выдачи инструмента

Автоматизированная система учёта выдачи инструмента в цехе металлоконструкций с использованием QR-кодов.

![Next.js](https://img.shields.io/badge/Next.js-16-black)
![Prisma](https://img.shields.io/badge/Prisma-6-blue)
![SQLite](https://img.shields.io/badge/SQLite-database-green)

## Возможности

### Терминал выдачи (кладовщик)
- 📱 Сканирование QR-кода сотрудника для авторизации
- 🔧 Сканирование QR-кода инструмента для выдачи/возврата
- 👤 Отображение списка инструментов на руках у сотрудника
- ⚠️ Предупреждение если инструмент выдан другому
- 📋 История операций за смену

### Панель администратора
- 👥 Управление сотрудниками (CRUD + печать QR-кодов)
- 🛠️ Управление инструментом (CRUD, фильтры, списание)
- 📁 Категории инструмента ("Обычный", "Особо ценный")
- 📊 Текущие выдачи с возможностью возврата
- 📈 Отчёты: активные/просроченные/история + экспорт CSV

## Технологии

- **Frontend**: Next.js 16, React 19, Tailwind CSS, shadcn/ui
- **Backend**: Next.js API Routes, Prisma ORM
- **Database**: SQLite
- **QR-коды**: qrcode.react

## Установка и запуск

### Требования
- [Node.js](https://nodejs.org/) 18+ или [Bun](https://bun.sh/)

### Шаг 1. Клонировать репозиторий

```bash
git clone https://github.com/drozdoff880508/tool-issuance-system.git
cd tool-issuance-system
```

### Шаг 2. Установить зависимости

```bash
bun install
```

### Шаг 3. Создать папку для базы данных

```bash
mkdir db
```

### Шаг 4. Инициализировать базу данных

```bash
bunx prisma db push
```

### Шаг 5. Заполнить базу тестовыми данными

```bash
bun run prisma/seed.ts
```

### Шаг 6. Запустить приложение

```bash
bun run dev
```

### Шаг 7. Открыть в браузере

Перейдите по адресу: **http://localhost:3000**

## Рабочий процесс

### Для кладовщика (терминал):

1. Открыть **http://localhost:3000/terminal**
2. Отсканировать QR-код сотрудника (или ввести код вручную)
3. Отсканировать QR-код инструмента
4. Нажать "Выдать" или "Принять возврат"

### Для администратора:

1. Открыть **http://localhost:3000/admin**
2. Добавить сотрудников и инструмент
3. Распечатать QR-коды (кнопка "QR-код" у каждой записи)
4. Наклеить QR-коды на пропуска и инструмент
5. Мониторить выдачи и просрочки в разделе "Отчёты"
